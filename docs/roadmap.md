# Roadmap

## Phase 1: Manual Note Capture

- Create notes manually in the browser app
- Store source URL, source title, selected text, raw thought, tags, and timestamps
- Generate deterministic placeholder structure fields
- List captured notes

## Phase 2: Browser Extension Selected-Text Capture

- Capture current tab URL and title
- Capture highlighted text from articles and PDFs where possible
- Send captured context into the existing note creation flow through the backend API route
- Keep extension settings local while leaving note persistence on the backend

## Phase 3: Voice-to-Note Capture

- Record short spoken thoughts
- Transcribe voice input
- Save transcribed thoughts as research notes with source context when available

## Phase 4: AI Structured Notes

- Replace fake structuring with real AI calls
- Extract summary, usefulness, purpose, claims, evidence, questions, and contradictions
- Keep raw thought and source context available for auditability

## Phase 5: Research Graph and Semantic Search

- Generate embeddings for notes and source snippets
- Add semantic search across research memory
- Connect notes by themes, claims, citations, and open questions
